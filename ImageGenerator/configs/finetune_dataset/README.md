## Fine-tuning dataset format (unified CLI)

The `finetune` command expects:

1. **`--data`**: path to a **single JSON file** that is an array of records.
2. **`--images_root`**: directory that **contains** the image files. Each record’s `image` path is relative to this (e.g. if `image` is `CLEVR_train_000000.png`, the file must be at `<images_root>/CLEVR_train_000000.png`). If your PNGs are in a subfolder (e.g. `images/`), set `--images_root` to that folder (e.g. `configs/finetune_dataset/images`).

Each record must have:

- `id` – record identifier (string or number).
- `image` – path relative to `--images_root` (e.g. `images/000001.png` or `CLEVR_train_000000.png`).
- `text` – caption / prompt for the image (with constraints if desired).

Example `dataset.json`:

```json
[
  {"id": "000001", "image": "images/000001.png", "text": "a photo of a cat on a sofa, photorealistic"},
  {"id": "000002", "image": "images/000002.png", "text": "a watercolor landscape with mountains and a lake"}
]
```

Example layout (if images live in a subfolder):

- `dataset.json` or `dataset_clip77.json` (the file passed to `--data`)
- `images/CLEVR_train_000000.png`, … (then use `--images_root configs/finetune_dataset/images` so that `image` value `CLEVR_train_000000.png` resolves to that file)

Example command (run from **ImageGenerator** project root). Use **one line** or put `\` at the end of each line so the shell does not treat each line as a separate command:

```bash
python -m src.common.cli finetune --model sd15 --data configs/finetune_dataset/dataset_clip77.json --images_root configs/finetune_dataset/images --out ckpts/sd15_lora --max_steps 50 --batch_size 1 --grad_accum 1 --seed 42
```

Train/val split is done inside the loader using `--val_ratio` (default 0.05).

### CLIP 77-token limit (SD 1.5)

SD 1.5 uses CLIP, which has a **77-token** limit per prompt. Long captions are truncated at train time. For CLEVR-style captions, a shortened version is provided:

- **`dataset_clip77.json`** – Same structure as `dataset.json`, but each `text` is a **shortened instruction** that fits in ≤77 CLIP tokens. Object list is always complete. Spatial relations (right/front) are included when they fit; in very large scenes only tail relation phrases may be omitted to stay under 77 tokens.
- Generated by (from project root):  
  `python3 scripts/compress_clevr_to_clip77.py configs/finetune_dataset/dataset.json configs/finetune_dataset/dataset_clip77.json`

Encoding used in `dataset_clip77.json` (shortest form may vary to fit 77 tokens):

- **Preamble:** `CLEVR 4 regions. N objects.` or `CLEVR 4r N objs.`
- **Objects:** e.g. `0: bl L rb cube r1` or shorter `0bl L rb cube 1` (index, color bl/gr/cy/br/gy/yl/pu/rd, size L/S, material met/rb, shape cube/sph/cyl, region r0–r3 or digit).
- **Relations:** `R0:2,5` or `R0,2,5` = right of object 0 are 2 and 5; `F0:1,3` = front of object 0 are 1 and 3.

Use `--data configs/finetune_dataset/dataset_clip77.json` (with the same `--images_root`) for SD 1.5 fine-tuning to avoid truncation.

**Inference:** If the model was fine-tuned on this compact style, use the **same style** at inference so prompts match. Convert long CLEVR prompts to compact with:

- **From code:**  
  `from src.common.clevr_compact import clevr_long_to_compact`  
  `compact = clevr_long_to_compact("Generate a CLEVR style image... There are 3 objects...")`
- **From CLI:**  
  `python3 scripts/prompt_to_compact.py "your long CLEVR prompt here"`  
  Or convert a JSONL file:  
  `python3 scripts/prompt_to_compact.py --jsonl configs/prompts_finetuned_test.jsonl -o configs/prompts_compact.jsonl`  
  Then use the output file as `--prompt_file` for `run_finetuned`. Prompts in the JSONL must be in long CLEVR form for conversion to apply; non-CLEVR lines are left unchanged (pass-through).
